📊 매출 데이터 자동 반영 — 매출대시보드 html 파일 업로드 시 전월매출·3개월평균·6개월평균 자동 갱신
매달 마감된 매출대시보드(html) 파일을 그대로 선택하면, 파일 안의 매출 데이터를 읽어서 점포명이 일치하는 매장의 전월매출·3개월평균·6개월평균·영업개월수를 자동으로 갱신합니다. 새로 생긴 매장(매출대시보드에는 있지만 여기 목록에 없는 경우)은 매칭 안 됨으로 안내되니, "신규 매장 추가"로 별도 등록해주세요. 반영 결과는 이 브라우저에 저장되며, 영구 반영하려면 하단 "전체 STORES 코드 내보내기"를 사용하세요.
➕ 신규 매장 추가 — 주소 입력 시 상권데이터·상권유형 자동 분석
상권유형은 이름이 아니라 위치 기반으로 자동 분류됩니다 (매장 추가 시 지도상 실제 반경을 분석). 우선순위: ① 지하철역 300m 이내 → 역세권 ② 대학교 500m 이내 → 대학가 ③ 법원·시청 등 관공서 300m 이내 → 오피스·관공서 ④ "종합병원" 명칭 300m 이내(개인 의원·동물병원 제외) → 병원상권 ⑤ 해당없음 → 일반상권. 분류 근거(거리·시설명)는 추가 완료 후 표에서 확인할 수 있습니다.
영업개월수는 오픈일자로 자동 계산됩니다. 오픈월은 영업일이 20일 이상일 때만 1개월로 인정됩니다 (예: 1/20 오픈 시 1월은 영업일 12일뿐이라 미포함, 2월부터 카운트). 이 개월수 기준으로 3개월 이하는 전월매출만, 3개월 초과 6개월 미만은 3개월평균까지, 6개월 이상은 전체 입력이 활성화됩니다.
추가한 매장은 이 브라우저에 자동 저장됩니다(새로고침해도 유지). 다른 컴퓨터·배포본에도 영구 반영하려면 "전체 STORES 코드 내보내기"로 복사한 코드를 index.html의 const STORES = [...] 부분에 붙여넣고 GitHub에 업로드하세요.
STORES 코드 (클립보드에 복사됨)
떠기다방 기존 점포 현황 (29개점)
이 표에 반영된 데이터: 층수·월세·면적은 수기 입력값이고, 상권유형·지하철거리·카페·저가형카페는 각 매장 실제 위치를 기준으로 카카오맵에서 자동 수집·분석한 값입니다. 상권유형은 지하철역(300m)〉대학교(500m)〉관공서(300m)〉종합병원(300m, 개인 의원 제외) 순으로 자동 분류됩니다 (뱃지에 마우스를 올리면 분류 근거 확인 가능). 이 항목들이 전부 "신규 입지 분석"의 유사매장 매칭·예상매출 계산에 그대로 쓰입니다 — 즉 여기 보이는 수치가 다른 탭 분석 결과의 실제 근거 데이터입니다.
점포명상권유형층수월세면적지하철거리카페저가형카페최근6개월평균최근3개월평균전월매출영업개월관리
영업개월수 기준: 3개월 이하는 전월매출만 표시, 3개월 초과는 3개월평균 추가, 6개월 이상은 6개월평균까지 표시됩니다 (안정적인 표본 크기가 될 때까지 평균을 보류하는 방식). 주황색 "(추정)" 표시는 아직 실매출이 없는 신규오픈 매장으로, 상권환경이 가장 비슷한 기존 매장 5곳의 평균으로 추정한 값입니다 — 실제 매출이 아니니 참고용으로만 활용하세요.
매장별 유사매장 매칭
이 화면이 보여주는 것: 각 매장을 실제 상권환경이 가장 비슷한 다른 매장들과 짝지어, 매출을 나란히 비교합니다. 단순히 "역세권끼리, 대학가끼리" 묶는 게 아니라, 아래 6가지 항목을 종합 채점(100점 만점)해서 점수가 높은 상위 3곳을 골라줍니다.

채점 기준: 상권유형 일치(25점) · 층수 일치(10점) · 면적 차이(최대 15점) · 지하철역 거리 차이(최대 20점) · 카페 밀집도 차이(최대 15점) · 저가형카페 밀집도 차이(최대 15점)
데이터 출처: 상권유형·층수·면적은 수기 입력값, 지하철거리·카페수·저가형카페수는 카카오맵에서 반경 500m 자동 수집한 실측값입니다.
활용법: 점수가 80점 이상이면 상권환경이 매우 유사한 매장이라 매출 격차의 원인(임차료·운영방식·경쟁강도 등 개별 요인)을 비교해볼 가치가 있고, 50점 미만이면 참고 정도로만 보시면 됩니다.
